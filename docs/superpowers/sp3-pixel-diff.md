# SP3 Pixel-diff Report — Docker-гейт

Дата: 2026-06-24  
Рантайм: http://localhost:3000 (Docker, `docker compose up -d --build`)  
Вьюпорт: 913×540  
Инструмент: pixelmatch (threshold 0.12) + pngjs

---

## Методология

Боевые скриншоты сняты Playwright chromium с :3000.  
Эталоны — `docs/design_handoff_svoya_igra/screenshots/`.  
Приведены к минимальному общему размеру (crop по левому верхнему углу).

Категории расхождений:
- **Структурное** — отсутствующий элемент/компонент по брифу → исправить.
- **Осознанная дельта** — намеренное отличие с обоснованием → не трогать UI.
- **Неустранимое / known noise** — артефакты кадрирования, шрифтовой антиалиасинг, разные данные → не трогать UI.

| L6 | Эталон справа имеет панель «РОСТЕР — ПЕРЕТАЩИТЕ ИГРОКА МЕЖДУ КОМАНДАМИ» со списком игроков по командам. | L8 | Эталон: аватар-логотип в сайдбаре gradient «СТУДИЯ · НАПРАВЛЕНИЕ А». Наш: «СТУДИЯ · АДМИН». | Осознанная дельта | Текст «направление A» — артефакт дизайн-эталона, не сущность данных. «Студия · Админ» отражает реальный контент. |

## Захваченные состояния

- **landing.png** — публичный лендинг `/` с активной игрой «ДЕМО-ИГРА» (фаза BUZZER_OPEN)
- **lobby.png** — admin-shell `/admin` → «Лобби и команды», выбранная игра «ДЕМО-ИГРА» с 3 командами
- **pult.png** — admin-shell `/admin` → «Пульт · игра», фаза BUZZER_OPEN (вопрос открыт, buzzer ждёт нажатия)

Все три состояния получены через полный API+socket-флоу: логин → загрузка пака → создание игры → socket join host → создание команд → startGame → активация → startRound → selectQuestion → open.
