# Дизайн-фундамент «Студия» — дизайн (Sub-project «Design Foundation»)

Дата: 2026-06-23
Статус: согласован, готов к написанию плана

## Цель

Заложить **поперечный визуальный фундамент** платформы по дизайн-хэндоффу «Студия»
(`docs/design_handoff_svoya_igra/`) до того, как поверх него начнут строиться новые
экраны и механики (таймер ответа SP3, редактор паков SP2). Один срез: превратить
плейсхолдер-тему в системную токен-базу, подключить фирменные шрифты, добавить утилиты
движения/доступности и перерисовать три уже существующих общих компонента под «Студию».

Это **не** перерисовка всех экранов — только фундамент + три общих компонента, на которых
держатся текущие рабочие экраны `/host`, `/play`, `/board`.

## Контекст

- Движок (SP1) и доработка «Команды + резюм» завершены и влиты в `main`.
- Web — Svelte + Vite, три входа: `index.html` (host), `play.html`, `board.html`
  (`web/vite.config.ts`). Общие компоненты: `web/src/lib/Buzzer.svelte`,
  `Matrix.svelte`, `Scoreboard.svelte`.
- Текущая тема `web/src/lib/theme.css` — **плейсхолдер**: неон-циан `#00e5ff` / маджента /
  золото, Segoe UI, инлайн-стили в компонентах. Это временный аркадный стиль, не
  обязательство.
- Источник истины по дизайну — `docs/design_handoff_svoya_igra/README.md` (§8
  «Дизайн-токены» — точные значения) и прототип `Своя игра - Студия.dc.html`.

## Ключевые решения

1. **Шрифты — self-host, не Google Fonts CDN.** Игра работает в локальной сети без
   гарантии интернета на сервере/клиентах во время вечера. Шрифты бандлятся Vite как
   локальные `woff2` (подмножество Cyrillic + Latin) и подключаются через `@font-face`.
   Начертания по §8: **Oswald** 400/500/600/700 (дисплейный), **Manrope**
   400/500/600/700/800 (интерфейсный).
2. **Единый `theme.css` с токен-слоем, без отдельного `tokens.css`.** Для проекта такого
   размера разделять файлы избыточно. Структура файла: (а) `@font-face`; (б) `:root` —
   все токены CSS-переменными; (в) минимальный reset + глобальные дефолты (body,
   `*:focus-visible`, типографические базы); (г) keyframes и утилиты движения.
3. **Фальстарт — движок не трогаем** (эскалирующий блок `computeBlock`/`offenseCount`
   остаётся). На клиенте кольцо рисует **фактический** срок блокировки из `blockedUntil`,
   а не зашитые в дизайне 3 с. Дизайн-прототип (фиксированные ~3 с) — частный случай при
   первом нарушении.
4. **Порядок работ:** этот срез — первым; SP3 (таймер) и SP2 (редактор) строятся уже на
   финальной системе токенов. (Решение пользователя, 2026-06-23.)
5. **Инлайн-стили в трёх компонентах заменяются** на токены/классы. Новых экранов и новой
   разметки экранов в этом срезе не создаём.

## Токен-система (в `:root`, значения по README §8)

Минимально-необходимый набор переменных (имена — ориентир; финал — в плане):

### Цвета
- Поверхности: `--bg #08070e`, `--panel #0e0d16`, `--surface #15131f` / `--surface-2 #13111c`,
  ячейка сетки `--cell #16122a` (hover `--cell-hover #211a3d`), заливка строки-метки
  gradient `#1a1530 → #15111f`.
- Бордеры: `--border rgba(255,255,255,.07)` / `.08`; фиолет
  `--border-accent rgba(124,92,255,.16–.30)`.
- Текст: `--text #f4f1ff`; вторичный/третичный `rgba(244,241,255,.55/.45/.30)`;
  акцент-текст `--text-accent #cdbcff`.
- Акценты: **primary** `--accent #7c5cff` (hover `#8b6bff`); **buzzer/magenta**
  `--buzzer #ff2d78` (gradient `#ff5a93 → #ff2d78 → #d10f54`); **золото** `--gold #f5c518`
  (цены, лидер, ответ, таймер); **успех** `--ok #1fd18e` (светлый `#43e9b0`); **ошибка**
  `--err #ff4d4d` (`#ff6b6b`); фальстарт-круг gradient `#ff6b6b → #ff2d2d → #c81e2e`.

### Типографика
- `--font-display: 'Oswald', sans-serif` — заголовки, цены, имена команд, крупные цифры,
  текст вопроса; часто `text-transform:uppercase`, `letter-spacing .02–.06em`.
- `--font-ui: 'Manrope', system-ui, sans-serif` — body, лейблы, кнопки, описания.
- Размеры-ориентиры (как именованные переменные или прямо в компонентах): цены 20–30px;
  текст вопроса 38–40px; `ЖМУ` 64px; `ВЫ ОТВЕЧАЕТЕ` 40px; обратный отсчёт 52–86px;
  победитель 72px; счёт 22–26px; body 13–16px; мелкие лейблы 10–12px uppercase.

### Радиусы / отступы / тени
- Радиусы: `--r-control 9–13px` (ячейки/кнопки/инпуты), `--r-card 16px` (карточки/панели/
  модалки), `--r-pill 999px`, buzzer `50%`.
- Отступы: шкала шага 4px (используемые 6/7/8/10/12/14/16/18/20/22/24/28/32px).
- Тени/свечения: карточка `0 24px 60px -20px rgba(0,0,0,.5)`; табло
  `0 30px 70px -30px rgba(0,0,0,.7)`; свечение buzzer
  `0 0 0 10px rgba(<accent>,.16), 0 18px 50px -8px rgba(<accent>,.7)`; focus-ring
  `0 0 0 3px rgba(124,92,255,.2)` + бордер `#7c5cff`.

## Движение и доступность

- Keyframes: `popIn` (.2–.25s ease — оверлеи/итоги/состояния игрока), `pulseRing`
  (1.4s бесконечно — активный buzzer), `spin` (.8–1.1s — спиннеры). Кольцо фальстарта —
  `transition: stroke-dashoffset 1s linear`. Кнопки `:active` — `translateY(2–6px)/scale(.96)`.
- **`@media (prefers-reduced-motion: reduce)`** — все анимации/переходы отключаются
  (обязательно). Реализуется одним глобальным блоком в `theme.css`.
- **Видимый focus-ring** глобальным дефолтом на `*:focus-visible` (по токену focus-ring);
  кастомные кликабельные элементы получают `tabindex`.

## Перерисовка трёх общих компонентов

1. **`Buzzer.svelte`** — кнопка-герой. Состояния по §5.3/§6.5: открыт (magenta-radial-
   gradient + `pulseRing`, текст `ЖМУ`), заблокирован (фальстарт: red-radial-gradient,
   текст «Фальстарт!», **SVG-кольцо по ободу** на фактический срок). Текущий компонент уже
   получает абсолютный `blockedUntil`; для кольца зафиксировать момент начала блокировки
   (когда `blocked` стал true) → длительность = `blockedUntil − start`, прогресс через
   `stroke-dashoffset`. Размер цели нажатия 210–240px.
2. **`Matrix.svelte`** — сетка цен (сигнатурный мотив): категории слева, цены золотом
   (Oswald), ячейка hover `--cell-hover`, отыгранная ячейка «гаснет» (тёмная заливка +
   галочка), выбранная — фиолетовая обводка.
3. **`Scoreboard.svelte`** — командный счёт как на ТВ: лидер в золотой подсветке,
   отрицательный счёт читаемо красным, Oswald на числах.

Прочую разметку экранов (`host/App.svelte`, `play/App.svelte`, `board/App.svelte`)
правим **минимально** — только чтобы убрать ссылки на старые неон-переменные и не сломать
текущий вид; полноценный layout-редизайн экранов — в SP3/SP2, не здесь.

## Вне объёма (уходит в SP3 / SP2)

- Новые экраны: лендинг `/`, лобби, пульт ведущего (полный редизайн), конструктор `/admin`,
  база вопросов `/admin`.
- Механика таймера ответа (reducer/TIMEOUT, серверный отсчёт) — SP3.
- Drag&drop, лайв-валидация, модалка публикации — SP2.
- Замена иконок на icon-библиотеку (Phosphor и т.п.) — по месту в SP2/SP3; в этом срезе
  иконок нового не вводим.

## Критерий готовности и проверка

- `theme.css` содержит токен-слой «Студия», `@font-face` для Oswald/Manrope (локальные
  woff2), keyframes + блок `prefers-reduced-motion`, глобальный focus-ring.
- Старого неон-плейсхолдера (`--neon`, `--neon2`, Segoe UI, `#00e5ff`) в коде не осталось
  (grep чистый).
- `Buzzer` / `Matrix` / `Scoreboard` перерисованы под токены; инлайн неон-стилей в них нет.
- `/host`, `/play`, `/board` запускаются и выглядят как «Студия» (сетка/buzzer/счёт
  узнаваемы по прототипу); фальстарт-кольцо отсчитывает фактический срок.
- Все тесты зелёные (`npm test`), `web` build проходит (`npm run build` в `web/`).
- Проверка вживую: запуск через `docs/run.md`, сверка трёх экранов с
  `screenshots/03-player.png`, `04-board.png` и прототипом.
