# Запуск и восстановление

## Запуск
```bash
docker compose up -d --build
```

## Роли и адреса

| Роль | Адрес |
|------|-------|
| **Игроки** | `http://<ip>:3000/` — лендинг с активной игрой; кнопка «Войти в игру» |
| **Ведущий** | `http://<ip>:3000/admin` — admin-shell |
| **Табло / ТВ** | `http://<ip>:3000/board?game=<gameId>` |

`<ip>` — локальный IP машины с контейнером (напр. `192.168.1.50`).  
На лендинге видна только **активированная** игра (одна за раз). Пока ни одна игра не активирована — лендинг показывает «Пока нет активной игры».

## Флоу ведущего

1. Открыть `http://<ip>:3000/admin` (пароль по умолчанию `admin`).
2. **База вопросов** — завести категории и вопросы (или импортировать пак через «Конструктор» → «Импорт»).
3. **Конструктор** — собрать игру (раунды, сетка вопросов) → **Опубликовать**.
4. **Лобби и команды** — выбрать опубликованный пак, задать название игры, нужное число команд → **Создать игру** → добавить/переименовать команды → **Активировать** (игра появится на лендинге у игроков).
5. **Пульт · игра** — ведение игры: выбор вопроса, открытие buzzer, судейство, переход между раундами → **Завершить игру** (игра исчезает с лендинга).

Ссылку на табло (`/board?game=<id>`) можно найти в секции «Сыграть тестовую» конструктора.

## Восстановление после сбоя
Состояние хранится в `./data` (SQLite + медиа), смонтированном как volume.

- **Перезапуск контейнера:** `docker compose restart` — игра продолжится с
  последнего события.
- **Сервер был на ноуте ведущего и ноут умер:** скопируй папку `data` на другую
  машину, запусти там `docker compose up -d`. Игроки переподключаются на новый
  IP, игра продолжается с того же места.

## Admin-панель

Живёт на `/admin` (за паролем):
- `http://<ip>:3000/admin`

Перед запуском задайте переменные окружения (через `environment:` в `docker-compose.yml`):
- `ADMIN_PASSWORD` — пароль администратора (плейн-текст). Дефолт: `admin`.
- `COOKIE_SECRET` — секрет для подписи сессионной куки. Для прод/LAN **обязательно задайте свои** значения.

### Конструктор игр

Раздел «Конструктор» (`/admin` → «Конструктор») собирает игру из вопросов банка. Поток:

1. **Создать игру** — «Новая 5×5» (готовая сетка 5 цен × 5 строк) или «Пустая игра».
2. **Собрать сетку раунда:**
   - сверху — столбцы-цены (золотые поля, кратно 100); кнопками **+ / −** меняется
     число столбцов, сетка остаётся прямоугольной;
   - слева у каждой строки — зона категории: перетащите категорию из сайдбара
     «База · источник» (привязка очищает ячейки строки);
   - в ячейки перетаскивайте вопросы — **только из категории этой строки** (чужой
     вопрос отклоняется красной рамкой);
   - клик по тегу в ячейке циклит мини-игру **нет → Кот → Аукцион**; крестик
     очищает ячейку;
   - «+ Раунд» добавляет раунд (вкладки сверху).
3. Изменения сохраняются автоматически (индикатор «Сохранение… → Сохранено ✓»).
4. **Плашка снизу** показывает, что мешает опубликовать (строк без категории,
   пустых ячеек); когда всё заполнено — зелёная «Всё заполнено — можно публиковать».
   Кнопка «Опубликовать» активна только при отсутствии ошибок.
5. **Сыграть тестовую** — публикует и открывает табло (`/board?game=<id>`).
6. **Опубликовать** — собирает пак. Если на этом паке есть активные игры,
   появится предупреждение: перезапись завершает эти игры.
