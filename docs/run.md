# Запуск и восстановление

## Запуск
```bash
docker compose up -d --build
```

## Роли и адреса

| Роль | Адрес |
|------|-------|
| **Игроки** | `http://<ip>:3000/` — лендинг с активной игрой; кнопка «Войти в игру» |
| **Ведущий** | `http://<ip>:3000/admin` — admin-shell |
| **Табло / ТВ** | `http://<ip>:3000/board?game=<gameId>` |

`<ip>` — локальный IP машины с контейнером (напр. `192.168.1.50`).  
На лендинге видна только **активированная** игра (одна за раз). Пока ни одна игра не активирована — лендинг показывает «Пока нет активной игры».

## Флоу ведущего

1. Открыть `http://<ip>:3000/admin` (пароль по умолчанию `admin`).
2. **База вопросов** — завести категории и вопросы (или импортировать пак через «Конструктор» → «Импорт»).
3. **Конструктор** — собрать игру (раунды, сетка вопросов) → **Опубликовать**.
4. **Лобби и команды** — выбрать опубликованный пак, задать название игры, нужное число команд, **«Время на ответ»** (10–120 с, дефолт 45 с) → **Создать игру** → добавить/переименовать команды → **Активировать** (игра появится на лендинге у игроков).
5. **Пульт · игра** — ведение игры: выбор вопроса, открытие buzzer, судейство, переход между раундами → **Завершить игру** (игра исчезает с лендинга).

Ссылку на табло (`/board?game=<id>`) можно найти в секции «Сыграть тестовую» конструктора.

### Таймер ответа

Когда команда нажала buzzer первой (фаза **ANSWERING**), на пульте и на экране игрока идёт обратный отсчёт. Номинал задаётся в Лобби при создании игры (поле «Время на ответ», 10–120 с).

- **По истечении:** ответ команды не засчитывается (−стоимость вопроса), ход автоматически переходит следующей команде в очереди buzz.
- **⏸ Пауза** (на пульте): замораживает отсчёт; число на экране не меняется.
- **▶ Продолжить** (на пульте): возобновляет отсчёт с того же момента.
- **↻ Сброс** (на пульте): перезапускает полный номинал для текущей отвечающей команды.

## Восстановление после сбоя
Состояние хранится в `./data` (SQLite + медиа), смонтированном как volume.

- **Перезапуск контейнера:** `docker compose restart` — игра продолжится с
  последнего события.
- **Сервер был на ноуте ведущего и ноут умер:** скопируй папку `data` на другую
  машину, запусти там `docker compose up -d`. Игроки переподключаются на новый
  IP, игра продолжается с того же места.

## Admin-панель

Живёт на `/admin` (за паролем):
- `http://<ip>:3000/admin`

Перед запуском задайте переменные окружения (через `environment:` в `docker-compose.yml`):
- `ADMIN_PASSWORD` — пароль администратора (плейн-текст). Дефолт: `admin`.
- `COOKIE_SECRET` — секрет для подписи сессионной куки. Для прод/LAN **обязательно задайте свои** значения.

### Конструктор игр

Раздел «Конструктор» (`/admin` → «Конструктор») собирает игру из вопросов банка. Поток:

1. **Создать игру** — «Новая 5×5» (готовая сетка 5 цен × 5 строк) или «Пустая игра».
2. **Собрать сетку раунда:**
   - сверху — столбцы-цены (золотые поля, кратно 100); кнопками **+ / −** меняется
     число столбцов, сетка остаётся прямоугольной;
   - слева у каждой строки — зона категории: перетащите категорию из сайдбара
     «База · источник» (привязка очищает ячейки строки);
   - в ячейки перетаскивайте вопросы — **только из категории этой строки** (чужой
     вопрос отклоняется красной рамкой);
   - клик по тегу в ячейке циклит мини-игру **нет → Кот → Аукцион**; крестик
     очищает ячейку;
   - «+ Раунд» добавляет раунд (вкладки сверху).
3. Изменения сохраняются автоматически (индикатор «Сохранение… → Сохранено ✓»).
4. **Плашка снизу** показывает, что мешает опубликовать (строк без категории,
   пустых ячеек); когда всё заполнено — зелёная «Всё заполнено — можно публиковать».
   Кнопка «Опубликовать» активна только при отсутствии ошибок.
5. **Сыграть тестовую** — публикует и открывает табло (`/board?game=<id>`).
6. **Опубликовать** — собирает пак. Если на этом паке есть активные игры,
   появится предупреждение: перезапись завершает эти игры.

### Как играть финал

Финальный раунд — особая фаза, где все оставшиеся команды одновременно делают ставки и пишут ответ на один общий вопрос.

#### Настройка таймера финала

В Лобби при создании игры есть поле «Таймер финала» (30–300 с, дефолт 60 с). Это время, которое даётся командам на обдумывание и запись ответа в финальной фазе «Вопрос».

#### Создание финального раунда в конструкторе

1. Откройте игру в «Конструкторе».
2. Нажмите кнопку **«+ Финал»** — она появляется рядом с «+ Раунд», если финального раунда ещё нет. Финал может быть только один.
3. В редакторе финального раунда задайте минимум **2 темы** (по умолчанию «Тема 1» и «Тема 2»). Кнопка «+ Тема» добавляет новую. Удалить тему нельзя, пока их всего 2.
4. Для каждой темы нажмите **«+ выбрать вопрос»** и выберите вопрос из банка. Каждая тема требует своего вопроса. Вопрос из финала не может быть использован в обычных раундах той же игры.
5. Плашка валидации покажет ошибки: «тем финала без вопроса N» или «в финале менее 2 тем». При исправлении всех ошибок кнопка «Опубликовать» становится активной.

#### Ход игры — финал

После завершения последнего обычного раунда в Пульте появляется кнопка **«Начать финал →»**. Поток:

1. **FINAL_INTRO — Вступление.** Ведущий видит список тем финала и назначает капитанов для команд с очками (выпадающий список по каждой команде). Затем жмёт **«Начать вычёркивание →»**.

2. **FINAL_ELIMINATION — Вычёркивание тем.** Команды по очереди (от наименьшего счёта) убирают по одной теме — пока не останется одна. Очерёдность вычёркивания — только команды с очками > 0. Ведущий видит, чья очередь; игрок-капитан жмёт нужную тему на своём экране.

3. **FINAL_BETTING — Ставки.** Каждая команда вводит ставку от 0 до своего текущего счёта. Ставка «тайная» — другие команды (и табло) не видят чужие ставки. Когда все сделали ставки — фаза переходит автоматически.

4. **FINAL_QUESTION — Вопрос и ответ.** Все видят вопрос оставшейся темы. Идёт обратный отсчёт (таймер финала). Каждая команда вводит ответ в своём окне и нажимает **«Готово»**. Ответы тайные — чужой текст скрыт. По истечении таймера все ответы фиксируются автоматически.

5. **FINAL_REVEAL — Вскрытие.** Ответы и ставки открываются по одной команде. Ведущий нажимает **«✓ Верно»** или **«✕ Неверно»** — счёт команды меняется на ±ставку. После всех команд становится активна кнопка **«Завершить игру»**.

**Примечание:** В финале участвуют только команды со счётом > 0 на момент старта (`startFinal`). Команды с нулевым или отрицательным счётом в вычёркивании и ставках не участвуют.

### Перенос игры между установками

«Скачать шаблон» в редакторе игры сохраняет файл `<название>.game.json` — это **только
компоновка** (сетка, цены, теги, ссылки на вопросы банка по UID), без медиа. «Импортировать
шаблон» в списке игр поднимает игру из такого файла.

Медиа и контент вопросов едут **отдельной веткой** — через «Экспорт банка» (`bank.zip`).
Чтобы перенести игру целиком на другую машину: сначала импортируйте банк (`bank.zip`),
потом шаблон (`*.game.json`). Если на целевой машине нужных вопросов в банке нет, ячейки
с висящими ссылками подсветятся как невалидные и публикация заблокируется до пере-привязки.
