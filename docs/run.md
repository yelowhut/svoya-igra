# Запуск и восстановление

## Запуск
```bash
docker compose up -d
```
- Ведущий: `http://<ip-сервера>:3000/` (или `/index.html`)
- Игроки: `http://<ip-сервера>:3000/play.html?game=<gameId>`
- Табло: `http://<ip-сервера>:3000/board.html?game=<gameId>`

`<ip-сервера>` — локальный IP машины с контейнером (напр. `192.168.1.50`).

## Восстановление после сбоя
Состояние хранится в `./data` (SQLite + медиа), смонтированном как volume.

- **Перезапуск контейнера:** `docker compose restart` — игра продолжится с
  последнего события.
- **Сервер был на ноуте ведущего и ноут умер:** скопируй папку `data` на другую
  машину, запусти там `docker compose up -d`. Игроки переподключаются на новый
  IP, игра продолжается с того же места.

## Админ-панель (редактор паков)

Админка живёт на `/admin` (за паролем):
- `http://<ip-сервера>:3000/admin`

Перед запуском задайте переменные окружения:
- `ADMIN_PASSWORD` — пароль администратора (плейн-текст).
- `COOKIE_SECRET` — секрет для подписи сессионной куки (любая длинная случайная строка).

В `docker-compose` пробросьте их через `environment:`. Без переопределения пароль
по умолчанию = `admin`, а секрет куки — фиксированная дев-строка: для прод/LAN
**обязательно задайте свои** значения.

### Конструктор игр

Раздел «Конструктор» (`/admin` → вкладка «Конструктор») собирает игру из вопросов
банка. Поток:

1. **Создать игру** — «Новая 5×5» (готовая сетка 5 цен × 5 строк) или «Пустая игра».
2. **Собрать сетку раунда:**
   - сверху — столбцы-цены (золотые поля, кратно 100); кнопками **+ / −** меняется
     число столбцов, сетка остаётся прямоугольной;
   - слева у каждой строки — зона категории: перетащите категорию из сайдбара
     «База · источник» (привязка очищает ячейки строки);
   - в ячейки перетаскивайте вопросы — **только из категории этой строки** (чужой
     вопрос отклоняется красной рамкой);
   - клик по тегу в ячейке циклит мини-игру **нет → Кот → Аукцион**; крестик
     очищает ячейку;
   - «+ Раунд» добавляет раунд (вкладки сверху).
3. Изменения сохраняются автоматически (индикатор «Сохранение… → Сохранено ✓»).
4. **Плашка снизу** показывает, что мешает опубликовать (строк без категории,
   пустых ячеек); когда всё заполнено — зелёная «Всё заполнено — можно публиковать».
   Кнопка «Опубликовать» активна только при отсутствии ошибок. Публикация
   дополнительно проверяет, что медиа-файлы вопросов физически на месте.
5. **Сыграть тестовую** — публикует игру и открывает экраны ведущего (`/host`) и
   табло (`/board`).
6. **Опубликовать** — собирает пак. Если на этом паке есть активные игры,
   появится предупреждение: **перезапись завершает эти игры** (играем сами, на
   несколько параллельных комнат не рассчитываем). По умолчанию предлагается
   «Новый пак».

> Связь живая: правка вопроса в банке отражается во всех играх до публикации; при
> публикации делается снимок (flatten в формат пака).
